Le fonctionnement est assez simple, chaque "type" (e.g. Organization) a :

  • un modèle dans /html = façon dont il sera affiché ;
  • un modèle dans /json-schema = façon dont les données sont représentées ;
  • des instances dans /json = les données d'Organization.

    • On peut ensuite visualiser chaque instance avec https:/skepticdb.github.io/?p=chemin_instance.

    Github permet à chacun de contribuer, ainsi que de gérer un historique des versions. En revanche, le PHP n'est pas supporté. La visualisation des données n'est donc pas vraiment performante (cela met un moment à s'afficher), le but principal est déjà de pouvoir accéder à des données formatées, il est ainsi possible de construire d'autres sites/fonctionnalités à partir des données JSON.
